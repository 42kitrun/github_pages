# TCP/IP, Web, HTTP 한눈에 이해하기

_written by Claude-Code,ChatGPT

TCP/IP, 웹(WWW), HTTP — 세 용어는 인터넷을 다루다 보면 반드시 마주친다. 그런데 "인터넷 = 웹"이라고 혼동하거나, HTTP와 TCP/IP를 같은 선상의 개념으로 보는 경우가 많다. 이 글에서는 세 개념의 **범위와 계층 관계**를 중심으로 혼동 없이 정리한다.

![TCP/IP, Web, HTTP 한눈에 정리|697](../../../assets/posts/cs/tcpip-http-web-infographic.svg)

## 첫 번째 원칙: 인터넷 ⊃ 웹 ⊃ HTTP

세 개념은 경쟁 관계가 아니라 **중첩된 포함 관계**다.

```
┌──────────────────────────────────────────────┐
│  인터넷 (Internet)                            │
│  TCP/IP 프로토콜 기반 · 전 세계 컴퓨터 네트워크│
│                                              │
│  이메일     FTP      DNS     SSH   스트리밍   │
│  (SMTP)  (파일전송) (도메인) (원격접속)(RTSP) │
│                                              │
│  ┌────────────────────────────────────────┐  │
│  │  웹 (WWW · World Wide Web)             │  │
│  │  하이퍼텍스트 기반 정보 서비스          │  │
│  │                                        │  │
│  │  ┌──────────────────────────────────┐  │  │
│  │  │  HTTP / HTTPS                    │  │  │
│  │  │  웹 데이터 통신 프로토콜          │  │  │
│  │  │                                  │  │  │
│  │  │  클라이언트 → GET /index.html     │  │  │
│  │  │  서버      ← 200 OK (HTML)        │  │  │
│  │  └──────────────────────────────────┘  │  │
│  └────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
```

- **인터넷**은 가장 바깥 범위다. 이메일, FTP, 스트리밍, 웹 — 모든 서비스의 토대다.
- **웹**은 인터넷 위에서 동작하는 서비스 중 하나다. 인터넷과 웹을 동일시하면 안 된다.
- **HTTP**는 웹 안에서 데이터를 주고받는 방식이다. 웹의 통신 규약이므로 인터넷 전체 프로토콜이 아니다.

---

## TCP/IP — 인터넷의 기반 규칙

TCP/IP는 인터넷 통신 전반을 지탱하는 **프로토콜 스택**이다. 단일 프로토콜이 아니라 계층별 규칙의 묶음이다.

| 계층 | 이름 | 주요 프로토콜 | 역할 |
|------|------|--------------|------|
| 4 | 애플리케이션 계층 | HTTP · SMTP · FTP · DNS · SSH | 사용자와 가장 가까운 계층 |
| 3 | 전송 계층 | TCP · UDP | 신뢰성 있는 송수신 |
| 2 | 인터넷 계층 | IP | 패킷 라우팅 · 주소 지정 |
| 1 | 네트워크 접속 계층 | Ethernet · WiFi · LTE | 실제 데이터 비트 전송 |

**IP**는 데이터를 패킷으로 쪼개 목적지 주소를 붙여 전송한다. 도착 순서나 손실은 보장하지 않는다.

**TCP**는 IP 위에서 동작하며 신뢰성을 보장한다. 3-way handshake로 연결을 수립하고, 손실된 패킷은 재전송, 순서가 뒤바뀐 패킷은 재조립한다.

```
Client          Server
  │──── SYN ────▶│
  │◀─── SYN+ACK ─│
  │──── ACK ────▶│  ← 연결 수립, 이후 데이터 교환 시작
```

TCP/IP는 웹만을 위한 것이 아니다. 이메일(SMTP), 파일 전송(FTP), DNS, SSH — 인터넷 위 모든 서비스가 TCP/IP 스택 위에서 동작한다.

---

## Web(WWW) — 인터넷 위의 하이퍼텍스트 서비스

웹은 1989년 팀 버너스리(Tim Berners-Lee)가 고안한 **하이퍼텍스트 기반 정보 공유 시스템**이다. 세 가지 핵심 기술로 구성된다.

| 구성 요소 | 역할 | 예시 |
|-----------|------|------|
| **URL** | 자원의 주소 | `https://example.com/page` |
| **HTML** | 문서 형식 | `<h1>제목</h1>` |
| **HTTP** | 요청/응답 규약 | `GET /page HTTP/1.1` |

브라우저(Chrome, Safari 등)는 웹의 클라이언트다. URL을 입력하면 HTTP 요청을 생성하고, 서버가 HTML을 반환하면 브라우저가 이를 렌더링해 화면에 표시한다.

> 인터넷 ⊃ 웹 — 이것이 핵심이다. 웹은 인터넷의 전부가 아니라, 인터넷이라는 도로망 위에 세워진 하나의 서비스다.

---

## HTTP — 웹의 통신 프로토콜

HTTP(HyperText Transfer Protocol)는 TCP/IP 스택의 **최상위 애플리케이션 계층**에 속하는 프로토콜이다. TCP 연결 위에서 동작하며, 브라우저와 서버 간 메시지 형식을 정의한다.

**요청(Request)**
```
GET /index.html HTTP/1.1
Host: example.com
Accept: text/html
```

**응답(Response)**
```
HTTP/1.1 200 OK
Content-Type: text/html

<html>...</html>
```

핵심 특징:
- **포트**: HTTP는 80, HTTPS는 443을 사용한다
- **HTTPS = HTTP + TLS**: 전송 데이터를 TLS로 암호화한 버전이다
- **상태 코드**: `200 OK` · `404 Not Found` · `500 Internal Server Error`
- **버전**: HTTP/1.1 → HTTP/2 → HTTP/3으로 발전하며 성능이 개선됐다

HTTP는 웹의 통신 방식이다. 인터넷 전체 프로토콜이 아니라는 점에서 TCP/IP와 구별된다.

---

## 실제 동작 흐름

주소창에 `https://example.com`을 입력하면 다음 순서로 동작한다.

```
① HTTP 요청 생성 (브라우저)
   GET /index.html HTTP/1.1

② TCP 3-way handshake → 서버와 연결 수립

③ IP 라우팅 → 패킷을 목적지까지 전달

④ 서버가 HTTP 응답 반환
   200 OK + HTML 문서

⑤ TCP가 응답 패킷 재조립

⑥ 브라우저가 HTML 렌더링 → 화면 표시
```

HTTP가 **무엇을** 주고받을지를 정의하고, TCP/IP가 그 데이터를 **어떻게** 목적지까지 전달할지를 담당한다.

## 세 개념 요약 비교

![TCP/IP, Web, HTTP 개념 정리](../../../assets/posts/cs/tcp-ip-web-http.png)

| 구분 | TCP/IP | 웹 (WWW) | HTTP / HTTPS |
|------|--------|----------|--------------|
| **정의** | 인터넷 통신의 기반 프로토콜 집합 | 인터넷 위의 하이퍼텍스트 서비스 | 웹의 데이터 통신 프로토콜 |
| **계층** | 전송·인터넷·링크 계층 | 응용 계층(서비스) | 응용 계층(프로토콜) |
| **범위** | 인터넷 전체 (웹·이메일·FTP 모두) | 인터넷 서비스 중 하나 | 웹 내 통신 방식 |
| **접근 방법** | 직접 노출되지 않음 | 브라우저로 접근 | 요청 → 응답 구조 |
| **오해 주의** | ≠ 인터넷의 유일한 규칙 | 인터넷 ≠ 웹 | ≠ 인터넷 전체 프로토콜 |
| **비유** | 도로 · 교통 법규 | 도로 위 건물 | 건물 안 안내 대화 방식 |

## 핵심 정리

세 개념의 관계는 **인터넷 ⊃ 웹 ⊃ HTTP**다.

- **TCP/IP**는 인터넷 통신의 토대다. 웹뿐 아니라 이메일, FTP, DNS 등 모든 인터넷 서비스가 사용한다.
- **웹(WWW)**은 그 위에서 동작하는 하이퍼텍스트 기반 서비스다. URL + HTML + HTTP로 구성된다.
- **HTTP**는 웹이 데이터를 주고받을 때 사용하는 통신 규약이다. TCP/IP 스택의 최상위 계층에서 동작한다.
