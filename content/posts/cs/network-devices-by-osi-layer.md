# 동작 계층으로 이해하는 네트워크 장비

_written by Claude-Code,ChatGPT_

네트워크 장비 이름은 익숙해도 각각이 정확히 어떤 역할을 하는지, 왜 허브 대신 스위치를 쓰는지, 라우터와 AP는 어떻게 다른지 물어보면 헷갈리는 경우가 많다. 이 글에서는 **OSI 계층을 기준**으로 각 장비의 동작 방식을 정리한다. 계층이 높아질수록 더 많은 정보를 보고 더 지능적으로 판단한다는 흐름으로 이해하면 전체 구조가 명확해진다.

![동작 계층 순서로 이해하는 네트워크 장비](../../../assets/posts/cs/network-devices-by-osi-layer.png)

---

## 계층이 핵심이다

네트워크 장비를 구분하는 가장 중요한 기준은 **어느 OSI 계층까지 데이터를 들여다보느냐**다.

```
계층 7 (응용)   │ 방화벽 (NGFW, WAF)       ← 트래픽 내용까지 분석
계층 4 (전송)   │ 방화벽 (상태 기반)
───────────────┼──────────────────────────
계층 3 (네트워크)│ 라우터, L3 스위치, AP     ← IP 주소로 판단
───────────────┼──────────────────────────
계층 2 (데이터링크)│ 스위치, 브릿지, AP       ← MAC 주소로 판단
───────────────┼──────────────────────────
계층 1 (물리)   │ 허브, 케이블, 리피터       ← 신호만 전달
```

계층이 낮을수록 단순하고 빠르다. 계층이 높을수록 더 많이 보고 더 정밀하게 제어한다.

---

## 계층 1 — 허브 (Hub)

허브는 물리 계층(Layer 1)에서 동작한다. 들어온 전기 신호를 **연결된 모든 포트에 그대로 복제해 내보낸다.**

- MAC 주소나 IP 주소를 전혀 인식하지 못한다
- 수신 대상이 누구든 모든 장치에 브로드캐스트한다
- 한 장치가 전송하면 나머지는 대기해야 한다 (충돌 도메인)

```
장치 A → 허브 → 장치 B, C, D 모두에게 전달 (의도와 무관하게)
```

현재는 보안·성능 문제로 거의 사용되지 않는다. 스위치로 완전히 대체됐다.

---

## 계층 2 — 브릿지 (Bridge)

브릿지는 데이터링크 계층(Layer 2)에서 동작한다. 허브와 달리 **MAC 주소를 학습해 필요한 포트에만 프레임을 전달한다.**

- 연결된 두 네트워크 세그먼트를 하나처럼 묶는다
- MAC 주소 테이블을 학습하며 불필요한 트래픽을 차단한다
- 포트 수가 적고 구조가 단순하다 — 스위치는 브릿지를 멀티포트로 확장한 형태다

```
세그먼트 A ── [브릿지] ── 세그먼트 B
              MAC 학습 후 필요한 방향으로만 전달
```

---

## 계층 2 — 스위치 (Switch)

스위치도 데이터링크 계층(Layer 2)에서 동작한다. 브릿지의 다중 포트 버전으로 이해할 수 있으며, 현대 LAN 구성의 핵심 장비다.

- **MAC 주소 테이블**을 유지하며, 목적지 MAC 주소가 연결된 포트로만 프레임을 전달한다
- 허브와 달리 각 포트가 독립적인 충돌 도메인을 가진다
- **VLAN(Virtual LAN)** 으로 하나의 스위치에서 논리적으로 네트워크를 분리할 수 있다

```
장치 A → 스위치 → MAC 테이블 조회 → 장치 C 포트로만 전달
                  (장치 B, D는 받지 않음)
```

허브가 "받은 것을 모두에게 나눠주는 것"이라면, 스위치는 "받아서 정확한 상대에게만 전달하는 것"이다.

---

## 계층 3 — 라우터 (Router)

라우터는 네트워크 계층(Layer 3)에서 동작한다. 스위치가 같은 네트워크 안에서 장치를 연결한다면, 라우터는 **서로 다른 네트워크를 연결하고 경로를 결정**한다.

- **IP 주소 기반**으로 패킷의 최적 경로(라우팅 테이블)를 계산해 전달한다
- **NAT(Network Address Translation)**: 사설 IP를 공인 IP로 변환해 인터넷에 연결한다
- **DHCP**: 네트워크에 새로 연결된 장치에 IP 주소를 자동 할당한다

```
[내부 네트워크 192.168.1.x]
장치 A (192.168.1.10)
장치 B (192.168.1.11)  ── [라우터] ── [인터넷 / 다른 네트워크]
장치 C (192.168.1.12)
```

가정에서 사용하는 **공유기**는 라우터 + 스위치 + AP + DHCP 기능을 하나로 통합한 장비다.

---

## 계층 2~3 — AP (Access Point, 무선 액세스 포인트)

AP는 데이터링크와 네트워크 계층(Layer 2~3)에 걸쳐 동작한다. **무선(WiFi) 클라이언트를 유선 네트워크에 연결하는 다리** 역할을 한다.

- 무선 장치의 MAC 주소와 IP 주소를 모두 처리한다
- 유선 이더넷과 무선 WiFi 사이의 브릿지다
- 기업 환경에서는 전용 AP 장비를, 가정에서는 공유기의 WiFi 기능이 이 역할을 수행한다

```
노트북 (WiFi) ── [AP] ── 유선 스위치 ── 라우터 ── 인터넷
스마트폰(WiFi) ──┘
```

AP는 단독으로 네트워크를 생성하지 않는다. 기존 유선 네트워크에 무선 접속 기능을 추가하는 장비다.

---

## 계층 4~7 — 방화벽 (Firewall)

방화벽은 전송 계층부터 응용 계층(Layer 4~7)까지 동작한다. 다른 장비들이 "어디로 전달할지"를 결정한다면, 방화벽은 **"이 트래픽을 허용할지 차단할지"를 결정**한다.

- **패킷 필터링**: IP, 포트, 프로토콜 기준으로 허용·차단한다
- **상태 추적(Stateful Inspection)**: 연결 상태를 기억해 비정상 패킷을 탐지한다
- **NGFW(Next-Generation Firewall)**: 응용 계층까지 분석해 앱별 트래픽 제어, 악성 URL 차단이 가능하다
- **WAF(Web Application Firewall)**: HTTP/HTTPS 트래픽을 분석해 SQL 인젝션, XSS 등을 차단한다

```
인터넷 ── [방화벽] ── 내부 네트워크
           ↑
    허용 정책에 맞는 트래픽만 통과
```

방화벽은 네트워크 경계에 위치하며, 내부와 외부 트래픽을 모두 감시한다.

---

## 실제 네트워크 흐름

노트북에서 웹 서버에 접속하는 과정을 장비 순서로 따라가면 다음과 같다.

```
노트북 (WiFi)
  │
  ▼ Layer 1~2
[AP / 스위치]  — 같은 LAN 안에서 MAC 주소로 프레임 전달
  │
  ▼ Layer 3
[라우터]       — IP 주소로 목적지 경로 결정, NAT 처리
  │
  ▼ Layer 4~7
[방화벽]       — 트래픽 허용 여부 검사
  │
  ▼
인터넷 → 웹 서버
```

계층이 올라갈수록 장비가 더 많은 정보를 보고 더 복잡한 결정을 내린다.

## 한눈에 비교

| 장비 | OSI 계층 | 주소 체계 | 주요 역할 | 사용 범위 |
|------|----------|----------|---------|---------|
| **허브** | 1 (물리) | 없음 | 신호 복제·전달 | LAN 내부 (구식) |
| **브릿지** | 2 (데이터링크) | MAC | 세그먼트 연결 | 소규모 세그먼트 간 |
| **스위치** | 2 (데이터링크) | MAC | LAN 내 장치 연결 | LAN 내부 |
| **라우터** | 3 (네트워크) | IP | 네트워크 간 경로 결정 | WAN·LAN 연결 |
| **AP** | 2~3 | MAC·IP | 무선 접속 제공 | 무선 네트워크 |
| **방화벽** | 4~7 (전송~응용) | 모든 주소 | 트래픽 허용·차단 | 네트워크 경계 |

---

## 핵심 정리

- **허브**는 신호를 모두에게 뿌린다. 스위치로 대체됐다.
- **스위치**는 MAC 주소로 정확한 상대에게만 전달한다. LAN의 핵심이다.
- **브릿지**는 두 세그먼트를 연결한다. 스위치의 원형이다.
- **라우터**는 IP 주소로 서로 다른 네트워크를 연결하고 경로를 결정한다.
- **AP**는 무선 장치를 유선 네트워크에 연결하는 다리다.
- **방화벽**은 트래픽의 허용·차단을 결정하는 보안 장비다.

계층이 높을수록 더 많이 보고, 더 정밀하게 제어한다. 이 원칙 하나로 여섯 장비의 차이가 명확해진다.
